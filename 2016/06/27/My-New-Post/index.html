Hexo+Github搭建个人博客

发表于 2016-06-27   |   分类于 git

hexo是一款基于Node.js的静态博客框架，它的界面简洁，有多种好看的主题供我们选择，通过hexo我们可以轻松地使用Markdown编写文章，还可以使用Hexo提供的标签插件来快速插入特定形式的内容。下面就一步步介绍Hexo搭建个人博客的过程。

环境配置

安装node.js

node.js的下载网址https://nodejs.org/en/

安装git

git的安装和配置可以参考http://www.cnblogs.com/zhcncn/p/3787849.html

hexo安装

安装

打开git Bash，执行如下命令：

npm install -g hexo

快速开始

(1).Setup your blog
Hexo安装完成后，在你喜爱的地方新建文件夹（如E：\Hexo）,在文件夹下面右键，选择Git Bash，执行如下命令，Hexo会自动在目标文件夹里建立网站所需要的所有文件。

hexo init

(2).Start the server
在（E:\Hexo下）运行下面的命令：
$ hexo server
表明Hexo Server已经启动，在浏览器中打开http：//localhost:4000/,就可以在本地预览页面效果，按住Ctrl+C停止Server。

(3).Generate static files

执行下面的命令将Markdown文件生成静态网页。
$ hexo generate

该命令执行完成后，在 E:\Hexo\public\ 目录下生成一系列html，css等文件。

配置Github

创建Resposity

在github中新建仓库，仓库名必须为【github用户名.github.io】

修改配置文件

在E：\Hexo文件夹下找到_config.yml文件，然后编辑该文件，将

   #deploy:
      type:
repository:
    branch:master

修改成：

  #deploy:

      type: git
repository: https://github.com/dongchunxia/dongchunxia.github.io.git
    branch:master

其中resporsitory是你刚刚创建的仓库的HTTPS,clone后粘贴上就可以了。

生成SSH Keys

我们如何让本地git项目与远程的github建立联系？这时候就要用到SSH Keys

（1）判断SSH keys是否存在

在Git Bash输入以下指令（任意位置点击鼠标右键），检查是否已经存在了SSH keys。

ls -al ~/.ssh

如果不存在就没有关系，如果存在的话，直接删除.ssh文件夹里面所有文件。

（2）生成SSH Keys

输入以下指令（邮箱就是你注册Github时候的邮箱）后，回车：

ssh-keygen -t rsa -C "2571451805@qq.com"

然后它会提示要你输入passphrase（如上图，我没有输入直接回车，如果你输入的话，要记得，到时候会用到）。

然后键入以下指令：

ssh-agent -s

输入之后，如果出现：

$ssh-add ~/.ssh/id_rsa
Could not open a connection to your authentication agent.

就输入下面的指令

eval `ssh-agent -s`
ssh-add

到了这一步，就可以添加SSH key到你的Github账户了。键入以下指令，拷贝Key（先拷贝了，等一下可以直接粘贴）：

clip < ~/.ssh/id_rsa.pub

然后到Github里面，点击右上角的设置图标，在Settings sidebar那里，点击SSH keys，点击Add SSH key，输入Title，作为这个key的描述，然后这个Key就是刚刚拷贝的，你直接粘贴就好。点击Add Key，输入你的Github密码即可完成SSH Key的添加。
键入以下命令来测试一下：

ssh -T git@github.com

你可能会看到有警告，没事，输入“yes”就好。

（3）部署到github
键入如下指令来完成部署：

$ hexo clean 
$ hexo generate
$ hexo deploy

当部署完成后，在浏览器中打开https://dongchunxia.github.io/，正常显示网页，表明部署成功。

常用命令总结

$ hexo new "postName" #新建文章
$ hexo new page "pageName" #新建页面
$ hexo clean #清除public下面的所有文件
$ hexo generate #生成静态页面至public目录
$ hexo server #开启预览访问端口（默认端口4000，'ctrl + c'关闭server）
$ hexo deploy #将.deploy目录部署到GitHub
$ hexo help  # 查看帮助
$ hexo version  #查看Hexo的版本
